# tlboard

AI가 제어할 수 있는 tldraw 화이트보드 CLI.

## 사전 조건

`tlboard serve`가 실행 중이어야 한다. 서버가 안 떠있으면 먼저 안내한다.

## 사용법 확인

매번 최신 사용법을 확인한다:

```bash
tlboard --help
```

## 기본 워크플로우

1. `tlboard status`로 서버 상태 확인
2. `tlboard shapes`로 현재 캔버스 내용 파악
3. `tlboard create '<json>'`으로 shape 생성
4. `tlboard screenshot`으로 결과 확인 (브라우저 필요)

## 실행 규칙

- shape 생성 시 `tlboard create`에 JSON을 넘긴다. 단일 객체 또는 배열.
- 여러 shape를 만들 때는 **배열로 한 번에** 생성한다 (API 호출 최소화).
- `connect`로 두 shape를 화살표로 연결할 수 있다.
- 결과 확인이 필요하면 `tlboard screenshot`으로 스크린샷을 찍어 확인한다.
- 브라우저(`http://localhost:3333`)가 열려있어야 screenshot이 동작한다.

## 다이어그램 레이아웃 규칙 (필수)

이 규칙을 지키지 않으면 결과물이 보기 어려워진다. 반드시 따른다.

### 여백 & 간격
- 박스 간 **최소 간격 30px** (가로, 세로 모두).
- 그룹 배경(큰 사각형)은 내부 박스보다 **상하좌우 15px 이상** 패딩.
- 행(row) 간 간격은 **50px 이상**.

### 박스 크기
- 텍스트가 잘리지 않도록 **w=200~220, h=50~60** 이상으로 설정.
- 텍스트가 2줄 이상이면 **h=80~100** 이상.
- 폰트 크기는 **`size: "s"`** 를 기본으로 사용.

### 화살표 (connect)
- **화살표가 다른 박스를 관통하거나 가리면 안 된다.**
- 화살표가 교차하면 레이아웃이 잘못된 것 — 배치를 바꿔서 교차를 없앤다.
- 화살표 대신 **텍스트 라벨 (예: `↓ HTTP`, `→ stdio`)** 로 관계를 표현하는 것이 더 깔끔할 수 있다.
- 화살표를 쓸 때는 **같은 열 또는 같은 행에서 1:1로 연결**한다.

### 겹침 금지
- 어떤 요소도 다른 요소와 **겹치면 안 된다** (박스, 텍스트, 화살표 모두).
- 그룹 배경 위에 박스를 올릴 때, 배경의 좌표와 크기를 계산해서 박스가 배경 안에 들어가는지 확인한다.

### 배치 패턴
- **수직 레이어 구조** (위→아래 흐름)를 선호한다.
- 같은 레이어의 요소는 **같은 y좌표에 수평 정렬**.
- 열은 **같은 x좌표에 수직 정렬** — 위아래가 대응되면 보기 좋다.
- 요소 수가 4개 이하면 1행, 5~8개면 2행으로 배치.

### 스크린샷 확인
- 다이어그램을 그린 후 반드시 `tlboard screenshot`으로 결과를 확인한다.
- 글자가 잘리거나, 겹치거나, 화살표가 관통하면 수정한다.

## 주의

- `tlboard --help`의 출력이 이 문서보다 최신일 수 있다. 항상 `--help`를 먼저 확인한다.
- shape ID는 서버가 자동 생성한다. `create` 결과에서 ID를 받아 `connect`/`delete`/`update`에 사용한다.
- 새 캔버스가 필요하면 `tlboard new "제목"`으로 만든다. 기존 캔버스는 `tlboard canvases`로 확인.
